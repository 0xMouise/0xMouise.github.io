黑盒测试的判断步骤

  1. 寻找疑似 SpEL 处理点

    • 在表单、接口、URL 参数中输入一些测试表达式，看返回内容或行为是否异常
      • 常见可疑功能包括：
        • 自定义规则表达式设置
        • 动态配置、模板变量填充
        • 使用 @Value()、Spring Security EL 的功能

  2. 使用探针 payload 检测漏洞

    • 检测型 payload（看是否被执行）

    • #{233+1}       → 看返回结果里有没有 234
${233+1}       → 某些也兼容
